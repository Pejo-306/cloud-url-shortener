# Project configuraiton

**NOTE:** Currently we only store backend configuration here.

Configuration parameters and secrets are stored in central `values.<environment>.enc.yaml`
files with secrets encrypted via `sops`. Encryption key pair is generated by `age` with 
private kye stored as a secret `SOPS_AGE_KEY` in GitHub environments.

Our [CD](../.github/workflows/cd.yml) decrypts these central files and generates
corresponding `config/**/<environment>.yaml` files needed by our bootstrap scripts
to seed SSM, Secrets Manager + generate AppConfig configuration.

**SECURITY NOTE:** The `age` private key is meant to be stored only as an environment secret
in GitHub environments. Since I don't want to create friction exploring this non-production
project, you can find the secret key in [key.txt](../key.txt) in plaintext.
THIS IS NOT SECURITY BEST PRACTICE!

## Generate local config files

To generate your own local config files for `make deploy` commands / bootstrap scripts:

1- Install `sops` and `age`

```bash
brew install sops age
```

2- Export `age` private key

```bash
export SOPS_AGE_KEY=AGE-SECRET-KEY-1T74R4ZJQJ5WAFHDTW6EV9F4F80YMC985QW4CUNAZWLQ4RCJFFLRSM6RCXJ
```

3- Decrypt into configuration files

```bash
sops -d config/values.dev.enc.yaml > config/shorten_url/dev.yaml
sops -d config/values.dev.enc.yaml > config/redirect_url/dev.yaml
```
